Comparison: Fetch vs. jQuery

In this lab, I used the Fetch API for weather and jQuery for time data.

Fetch API: I liked using Fetch with async/await. It makes the code look very
clean and easy to read. However, Fetch doesn't catch HTTP errors (like 404)
automatically. I had to write extra code using if (!response.ok) to handle
errors correctly. It is also good for adding timeouts with AbortController.

jQuery AJAX: jQuery is very fast to write. Using .done() and .fail() is simpler
for small tasks because it's built into one chain. It feels older than Fetch but
is still very reliable for quick API calls.

My Preference: I prefer Fetch. It is built directly into the browser, so I don't
need to load extra libraries. It feels like the modern way to write JavaScript.
Even though it needs more lines for error checking, it gives more control over
the request.
